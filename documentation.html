Project Overview

The Rizal Shrine Dapitan City website is a modern, responsive historical landmark information portal. It showcases the heritage, attractions, and visitor information for the Rizal Shrine located in Dapitan City, Zamboanga del Norte, Philippines.

Key Features

  • Responsive design optimized for desktop, tablet, and mobile devices
  • Modern glass-morphism UI design
  • Smooth scroll behavior and animations
  • SEO-friendly HTML structure
  • Docker support for containerization

Responsive Design Breakpoints

The website uses mobile-first responsive design with the following breakpoints:

Breakpoint Device Type Adjustments
max-width: 480px Small phones Compact navbar, smaller buttons, adjusted typography
max-width: 768px Tablets & large phones Single column layout, mobile menu toggle
max-width: 1024px Small desktops Two-column grids
1024px+ Desktop Full multi-column layout

Key Components

Navbar (.navbar)

  • Fixed positioning with glass-morphism effect
  • Collapsible mobile menu toggle at 768px
  • Smooth scroll behavior triggers .scrolled class
  • Logo with image and text branding

Hero Section (.hero)

  • Full viewport height (100vh)
  • Background image with overlay effect
  • Centered content with glass-morphism card
  • Call-to-action buttons

Cards (.card)

  • Glass-morphism styling with backdrop blur
  • Image, title, and description structure
  • Hover effects and animations

Buttons (.btn)

  • .btn-primary — Primary action buttons (solid background)
  • .btn-outline — Secondary action buttons (transparent with border)
  • Responsive padding adjustments for mobile

Setup & Deployment

Local Development

  1. Clone the repository
  2. Open index.html in your browser
  3. No build process required for development

Docker Deployment

Build and run the Docker container:

docker build -t rizal-shrine .
docker run -p 8080:80 rizal-shrine

Or use Docker Compose:

docker-compose up

Access the site at http://localhost:8080

Optimization Tips

  • Images are loaded via external CDN or local /images folder
  • Google Fonts are preloaded for faster rendering
  • CSS is minified in production (optional)
  • JavaScript uses vanilla JS for minimal bundle size

Browser Support

  • Chrome/Edge 90+
  • Firefox 88+
  • Safari 14+
  • Mobile browsers (iOS Safari, Chrome Mobile)

Notes for Developers

  • No frameworks: This is vanilla HTML, CSS, and JavaScript for simplicity and performance
  • SEO-friendly: Proper semantic HTML, meta tags, and Open Graph tags for social sharing
  • Accessibility: Semantic HTML and ARIA labels improve accessibility
  • Performance: Minimal CSS, efficient animations, lazy loading for images
